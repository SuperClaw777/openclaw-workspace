# HEARTBEAT.md - Kimi Claw 自我迭代系统

## 触发频率
每 30 分钟执行一次检查

## 检查清单

### 1. 任务队列检查
- 读取 `memory/tasks.json`
- 找出状态为 pending 的任务
- 根据优先级和时间安排执行

### 2. 项目状态检查
- 检查活跃项目的文件变更
- 查看 git 状态
- 记录进展到 `memory/progress/YYYY-MM-DD.md`

### 3. 待办提醒
- 检查是否有即将到期的任务
- 提醒用户（通过 Telegram）

### 4. 自我总结
- 每 8 小时生成一次工作摘要
- 更新 `memory/worklog.md`

### 5. 自动执行条件
以下情况自动触发工作：
- 有 pending 任务且超过 1 小时未处理
- 用户明确标记为 urgent 的任务
- 定时任务（如每日备份、日志清理）

## 执行规则
- 简单任务（<5分钟）：直接执行
- 复杂任务（>5分钟）：派生子代理
- 需要确认的任务：先询问用户

## 输出
- 发现问题 → 记录到 `memory/issues.md`
- 完成任务 → 更新 `memory/tasks.json` + 通知用户
- 需要决策 → 发送 Telegram 消息
